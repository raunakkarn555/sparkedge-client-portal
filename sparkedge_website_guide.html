Your Website
Management Guide

Everything you need to keep your SparkEdge website updated, running, and looking great — explained in plain English. No tech background needed.

Before you start: Your website is built in pure HTML, CSS, and JavaScript, and hosted free on GitHub Pages. Think of it as a folder of organised text files. Most updates are as simple as changing a word in a document — you just need to know which file and where to look.

Section 01
How to Update Text Content
Change headings, paragraphs, phone numbers, service descriptions — any visible words on your site.
1
Open your GitHub repository
Go to github.com and sign in. Click on your repository (it will be named something like your-username/your-website). SparkEdge would have shared this link with you at delivery.
2
Find the right file
Your website is made up of a few .html files — one for each page. You'll see them listed in the file list. Common ones are:
index.html ← Home page
about.html ← About page
services.html ← Services page
contact.html ← Contact page
Click whichever page has the text you want to change.
3
Open the editor
Once you've clicked the file, you'll see its contents. Click the pencil icon ✏️ in the top-right of the file view. This opens a simple text editor right inside the browser — no downloads needed.
4
Find your text using Ctrl+F (or Cmd+F on Mac)
Use the browser's Find function to search for the exact words you see on your website. For example, if your site says "We build websites for schools", search for that phrase. It'll highlight exactly where it lives in the code.
Tip: The text you see on screen is always the same text in the file — it just has a little HTML around it, like <h1>Your Heading</h1>. Only change the words between the tags — leave the <angle brackets> alone.
5
Change only the words — not the surrounding code
Here's an example of what safe editing looks like:
BEFORE:
<h1>We build websites for schools</h1>

AFTER (safe ✓):
<h1>We build award-winning websites for schools</h1>

AFTER (unsafe ✗ — don't remove the tags!):
We build award-winning websites for schools
6
Save your changes
Scroll to the bottom of the editor page. You'll see a green "Commit changes" button. Click it. Your site will update automatically within 1–2 minutes. Refresh your website in a new tab to see the changes live.
Don't worry — if something looks wrong, just go back into the file and change it back. GitHub keeps a history of every version so nothing is permanently lost.
Section 02
How to Replace an Image
Swap out a photo, logo, or banner without breaking the layout.
1
Prepare your new image file
The most important rule: name your new image exactly the same as the old one. For example, if the current image is called banner.jpg, your new file must also be saved as banner.jpg.
File naming rules: Use only lowercase letters, hyphens, and numbers. No spaces. No special characters.
✓ team-photo.jpg ✗ Team Photo (final).JPG
2
Find your images folder on GitHub
Inside your repository, look for a folder called images or assets (SparkEdge will confirm which one your site uses). Click into it to open the folder.
3
Upload your new image
Inside the images folder, click the "Add file" → "Upload files" button. Drag your new image in, or click to browse. Then click the green "Commit changes" button to save.
Uploading a file with the same name automatically replaces the old one. Your website layout stays exactly the same — only the photo changes.
4
Recommended image sizes
Keep images web-optimised so your site stays fast.
Hero / Banner: 1200 × 600 px · Under 300 KB
Team / Profile: 600 × 600 px · Under 120 KB
Logo: PNG with transparent bg · Under 80 KB
Gallery photo: 800 × 600 px · Under 200 KB
Use squoosh.app (free, in your browser) to compress images before uploading.
Section 03
Update Contact Information
Change your phone number, email address, or WhatsApp link across the whole site.
1
Phone numbers and email addresses
Open your contact.html file (and index.html if the number also appears on the home page). Use Ctrl+F to search for your old number — e.g. search 9876543210. Change every instance you find to your new number.
2
WhatsApp button link
Your WhatsApp button looks like this in the code. Find it and update only the number (the 10 digits after the country code 91):
BEFORE:
<a href="https://wa.me/919876543210">WhatsApp Us</a>

AFTER (new number):
<a href="https://wa.me/919999900000">WhatsApp Us</a>
Always keep the https://wa.me/91 part exactly as-is. Only change the 10-digit number after it.
3
Email address
Email links in the code look like this — update both the link target and the visible text:
<a href="mailto:info@yourbusiness.com">info@yourbusiness.com</a>
Change the address in both places — inside the quotes AND between the tags.
4
Check all pages, not just one
Contact details often appear in the footer, which is copied across every page. Make sure you update the phone/email in every .html file in your repository. A quick Ctrl+F across each file will confirm.

Section 04
How to Check if Your Site is Live
Quickly verify your website is accessible and loading correctly for visitors.
1
Visit your website address
Open any browser (Chrome, Safari, etc.) in a private/incognito window and type your website address. A private window ensures you're seeing the live site, not a saved version from cache.

Your website URL will look like one of these:
GitHub Pages (free hosting):
https://your-username.github.io/your-site-name

Custom domain (if you purchased one):
https://yourbusiness.com
2
Check the GitHub Pages status
On GitHub, go to your repository → click "Settings" → scroll down to "Pages" in the left sidebar. You'll see a message like:
✓ Your site is live at https://username.github.io/site
After you make any change and commit it, the site can take 1–3 minutes to update. If you don't see changes immediately, wait a moment and do a hard refresh (Ctrl+Shift+R or Cmd+Shift+R).
3
Test on mobile too
Open your website on your phone's browser to confirm it looks good on mobile. Your SparkEdge site is built mobile-first, so it should look great — but a quick check never hurts.
Section 05
How to View Basic Traffic Data
See how many people are visiting, which pages they read, and where they come from.
1
Google Analytics (if set up for you)
If SparkEdge set up Google Analytics for your site, visit analytics.google.com and sign in with the Google account linked to your site. You'll land on a dashboard showing:
Active users right now ← real-time visitors
Users (last 28 days) ← total unique visitors
Sessions ← total visits
Top pages ← which pages get most views
Traffic source ← Google search, direct, social
2
Google Search Console (for search rankings)
Visit search.google.com/search-console. This shows you which keywords people typed to find your site, how many times your site appeared in search results, and whether Google has found any issues.
If you don't have Analytics or Search Console set up yet, ask SparkEdge to add them — it takes about 15 minutes and is completely free.
3
Quick check without any tools
Simply open your browser and type your business name into Google. If your website appears in the results, it's indexed and working. A result appearing in the top 10 is a great sign your SEO is doing its job.
Section 06
What NOT to Touch
These areas are the engine under the bonnet — safe to leave alone, easy to accidentally break.
Leave These Exactly As-Is
CSS Variables
Lines that look like --primary-color: #FF6B2B; inside a <style> block. These control your entire colour scheme. Changing one number can shift every button, heading, and card colour on the site at once.
<script> tags
Any code between <script> and </script> tags. These power animations, menus, forms, and interactive features. Even one missing character can stop a form from working or break the mobile menu.
Structural HTML
Tags like <head>, <nav>, <section>, <footer>. These are the skeleton of the page. Moving or deleting them will collapse the layout.
CSS Stylesheets
Your .css files (e.g. style.css) contain hundreds of design rules. Don't edit these unless SparkEdge has specifically guided you on what to change.
Analytics codes
Snippets that look like G-XXXXXXXXXX or long blocks of Google/Meta code in the <head>. These track your visitors. Removing them means losing all traffic data.
File / folder names
Never rename any existing .html, .css, or .js file, and never rename the images folder. Other files reference each other by name — renaming one breaks the connection and causes 404 errors.
Golden rule: If you're not 100% sure what something does, don't touch it. When in doubt, take a backup first (see Section 08), then make the change. Or simply ask SparkEdge — there are no silly questions.

Section 08
How to Take a Backup
Download a complete copy of your website files to your computer — takes less than 2 minutes.
Open your repository
›
Click "Code" button
›
Download ZIP
›
Save to your computer
1
Go to your GitHub repository
Open github.com, sign in, and click on your website repository to open it.
2
Click the green "Code" button
Near the top-right of your file list, you'll see a green button that says "<> Code". Click it to open a small dropdown menu.
3
Click "Download ZIP"
At the bottom of that dropdown, you'll see "Download ZIP". Click it. Your browser will download a .zip file containing every file in your website — images, HTML, CSS, everything.
4
Save it somewhere safe and label it clearly
Move the downloaded ZIP to a folder you'll remember — like Google Drive or a "Website Backups" folder. Rename it with today's date so you can tell versions apart.
Good naming examples:
website-backup-jan-2025.zip
mysite-before-update-15feb2025.zip
GitHub actually keeps a full version history of every change you've ever made — so you have a built-in backup already. But downloading a ZIP gives you an offline copy just in case. We recommend doing this once a month, or before any major change.
