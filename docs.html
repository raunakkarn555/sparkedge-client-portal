SparkEdge Docs — Complete Workflow Guide
From first client call to final handover — step by step
🧭 The Golden Rule
Every client interaction follows a document flow. Never start work without an advance — never send advance invoice without a proposal. The sequence below is non-negotiable.
1
Discovery Call → Generate Proposal
After speaking with the client: select their service category (Web / Billing / Design / Landing Page), choose the Proposal document, fill in their details, preview and download. Share the HTML file via WhatsApp or email.
📊 Proposal Send BEFORE any payment
2
Client Confirms → Generate Invoice (Advance)
Client says yes? Generate the Invoice. Set Payment Type to Advance (50%), Payment Status to Pending. Share it. Once payment received, change status to Paid, enter Transaction Ref ID and re-download.
🧾 Invoice Send AFTER advance paid
3
Kickoff → Generate SOW / Agreement
Once advance is received and work begins: generate the SOW / Agreement (or Billing Setup Agreement). This legalises the scope and protects both parties. Mention pages, extras, and delivery timeline clearly.
📋 SOW / Agreement
4
Kickoff → Welcome Packet (Web / LP clients)
For Web Design and Landing Page clients, also send the Welcome Packet. It sets expectations, lists deliverables, and communicates timelines. For Landing Page clients, also generate and send the Content Brief — this is the form the client fills with their content, tagline, CTA, and images.
🎉 Welcome Packet 📝 Content Brief (LP only)
5
Delivery → Final Invoice + Delivery Note
Work is done! Generate the Invoice again but now with Final (50%) payment. Once final payment received, generate the Delivery / Handover Note. Add the live URL, Google Drive link, and any revision notes.
🧾 Final Invoice 📦 Delivery Note
💾 Pro Tip: Always Save Drafts
Use 💾 Save Draft for every client. Name it "ClientName — Document". This lets you re-open, edit and regenerate without re-filling. Export drafts regularly and share with the team.

Editing Fixed Document Content
When and how to override default text in documents
What is "Fixed Content"?
Some parts of every document are hardcoded defaults — the included features list, owner name, email, UPI ID, the "What's Included" checklist, "Not Included" items, support period, etc. These are locked to standard values so every doc looks professional. But sometimes a specific client needs different inclusions or a custom message.
How to edit fixed content for a specific client:
1. Click the ⚙️ Edit Fixed Defaults button in the top nav bar.
2. A modal opens with all editable fixed content fields for the CURRENT document type.
3. Change whatever you need — inclusions, deliverables, owner contact, UPI, support period, etc.
4. Click "Apply & Refresh Preview". The preview updates instantly.
5. Download your customised document.
6. These overrides are session-only — they reset when you reload. For permanent changes, update the source HTML file.
⚠️ Common use cases:
• Client gets a custom package (more pages, extra revisions) → Edit "What's Included"
• Client negotiates different payment split → Edit advance/final % defaults
• Different UPI for a specific deal → Edit UPI ID
• Team member sends doc from their email → Edit owner name/email
• Billing client needs different "Not Included" list → Edit that field
