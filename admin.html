System Guide
Complete explanation of how the SparkEdge Client Portal system works — from admin to client view.
🏗️
The Architecture
The SparkEdge portal is a static HTML system hosted on GitHub Pages — no database, no server, no monthly fees. It consists of 3 core files and a client folder:

• index.html — The login page where clients enter their ticket ID (e.g. SE-001) to access their dashboard.
• admin.html — This file. Your private workspace for managing everything. Not hosted publicly (or password-protected).
• docs.html — Document Generator. Creates Invoice, SOW, Welcome Packet, and Proposal as downloadable HTML/PDF files.
• clients/SE-001.html, SE-002.html etc. — Individual client dashboards, generated from admin and pushed to GitHub.
🎫
The Ticket System
Each client has a unique Ticket ID (e.g. SE-001, SE-002). When a client enters this ID on the login page, they're redirected to their personal dashboard at clients/SE-001.html.

The ticket stores: Client identity, project status & progress, payment stages, deliverables checklist, timeline updates, project files, and a note from the manager.

To add a new client: Go to New Ticket → Fill details → Save → Generate & Download the file → Put it in clients/ folder → Also add the ticket ID to the validTickets array in index.html → Push to GitHub.
💾
Data Storage
Admin data (tickets, vouchers, team members, settings) is stored in your browser's localStorage on your device. This means:

• ✅ Works offline, no server needed.
• ✅ Fast, instant saves.
• ⚠️ Data is per-device and per-browser. If you clear browser data, tickets may be lost — always generate & save the HTML files to GitHub as your backup.
• ⚠️ If you switch devices, re-create tickets or import from saved HTML files.

Client-facing files (the clients/SE-001.html files) are the real source of truth once pushed to GitHub.
💰
Payment & Discount System
Payment Stages: Add multiple stages (Advance, Milestone, Final) with amounts and statuses (paid / pending / upcoming). These show on the client's dashboard with visual indicators.

Situational Discount: Applied manually by you when warranted (festival offer, repeat client, etc.). Set flat ₹ or percentage. The discounted total is shown in the client file.

Referral Discount: Applied via the Referrals page. When a client's voucher code is redeemed by a new client, both get credited automatically. The new client's ticket shows a "Referral Discount Applied" banner, and the referrer's ticket shows a "Your Referral Rewards" section.
📄
Document Generator
The Doc Generator creates 4 types of professional documents:

1. Invoice — Formal payment document. Fill client info, services rendered, amounts, payment status, due date. Download → Open in browser → Print as PDF.

2. SOW / Agreement — Scope of Work and legal agreement. Protects you from scope creep. Both parties "agree" by proceeding.

3. Welcome Packet — Warm onboarding document explaining the process, your response times, revision policy, and contact info.

4. Client Proposal — Sales document sent before a client commits. Covers what they get, timeline, investment, and a confirmation section.
🎁
Referral System
How it works end-to-end:

1. You create a Referral Voucher in the Referrals page, assign it to a client (the referrer), set a discount for the new client, and specify the referrer's reward.
2. The referrer shares their code (e.g. SE-REF-ROHAN) with friends.
3. A new client signs up and gives you the code → You go to Referrals → Redeem Code → Enter the code and the new client's ticket ID → Confirm.
4. Both are updated: new client gets the discount shown on their portal; referrer gets a timeline entry and reward note on their portal.
5. You can also apply a referral discount directly in the ticket editor under Payment.
🔗
How to Edit the Doc Generator Button Link
The Doc Generator ↗ button in the sidebar currently opens docs.html. If your file is hosted on GitHub Pages or at a different URL, you may need to update this link.

To change the link:
1. Open admin.html in any text editor (VS Code, Notepad, etc.).
2. Press Ctrl+F (Cmd+F on Mac) and search for: docs.html
3. You will find this line in the sidebar navigation section:
window.open('docs.html','_blank')
4. Replace docs.html with your full URL, for example:
window.open('https://yourusername.github.io/yourrepo/docs.html','_blank')
5. Save the file. The button will now open the correct URL.

Similarly, the Client Portal ↗ button uses index.html — update it the same way if you move to a custom domain or different path.
🚀
Deploying to GitHub Pages
1. Make sure your repository has all files: index.html, admin.html, docs.html, assets/ folder, and clients/ folder.
2. Go to GitHub → Repository Settings → Pages → Source: Deploy from branch → Branch: main → Folder: / (root).
3. Your site is live at https://yourusername.github.io/yourrepo/.
4. Every time you generate a new client file: move it to clients/, run git add . && git commit -m "update" && git push. Live within 60 seconds.
5. Share https://yourusername.github.io/yourrepo/ as the portal link. Clients enter their SE-00X ticket ID to access their dashboard.
